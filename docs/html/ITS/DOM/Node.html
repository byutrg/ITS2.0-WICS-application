NAME

ITS::DOM::Node - thin wrapper around underlying XML engine node objects

SYNOPSIS

    use ITS::DOM;
    use feature 'say';
    my $dom = ITS::DOM->new(xml => 'path/to/file');
    my @nodes = $dom->get_xpath('//@foo');
    for my $node(@nodes){
        say $node->text;
    }

DESCRIPTION

This module is meant for internal use by the ITS::* modules only. It is a thin wrapper around an XML::LibXML::Node.

METHODS

new

Argument: a single XML::LibXML node.

Note that this constructor DOES NOT support subclassing by other libraries. The package of the returned object depends on the type of XML::LibXML node passed in.

to_string

Returns a string representation of this node.

get_xpath

Constructs an xpath query from the input arguments, and returns a list of nodes matching the query.

The xpath context node will be the calling node, and the first argument should be the XPath string. The rest of the parameters are named and optional:

position

An integer indicating the context position. Default is -1.

size

An integer indicating the context size. Default is -1.

params

A hash ref containing variable names and string values. No other types of values are allowed. There are no default parameters.

namespaces

A hash ref of namespace prefix keys and namespace URI values to be made available to the XPath expression. Any previously scoped namespaces are removed (By default, all namespaces in the scope of the context node are available, but providing a namespaces value removes these.).

type

Returns a string representing the type of the node: ELT, ATT, TXT, NS, PI, COM or DOC.

name

Returns the name of the node. This is the tag name for elements, the name for attributes and PIs, etc.

set_name

Sets the node's name to the given string. Is namespace aware.

remove

Unbinds this node from its siblings and parents (but not the document; though it becomes hidden and will be lost unless pasted somewhere in the document).

namespace_URI

Returns the namespace URI of this node. If there is no namespace associated with this node, returns an empty string.

local_name

Returns the node name without the namespace prefix.

value

Returns a string representing the value of the node. This is not text content for elements (call text for that).

text

Returns the content of all text nodes in the descendants of this node.

get_namespaces

Returns a hash ref containing prefix/URI pairs for all of the namespaces in scope for this node.

children

Returns a list containing the child nodes of this node.

parent

Returns the parent node of this node, or undef if there is none.

doc_node

Returns the document node for the document which owns this node, or undef if there is no owner.

path

Returns an XPath uniquely identifying this node in the current document.

next_sibling

Returns the next sibling node of this node, or undef if there is none.

prev_sibling

Returns the previous sibling node of this node, or undef if there is none.

paste

Paste this node into the given relation with the given node. The arguments are another node, and an optional relation. The possible relations are last_child, first_child, before and after. The default is last_child.

append_text

Appends the input text in the document relative to this node, and returns the new text node.

The arguments are the string to append, and an optional relation to specify where to paste the text relative to this node. The possible relations are the same as for paste, and the default is last_child.

is_same_node

Return true if this node is the same node as the input node.

copy

Returns a copy of this node. A single boolean argument indicates whether a deep copy should be performed (true for yes); that is, whether children should also be copied.

unique_key

Returns a unique value guaranteed to always be the same for this node. In other words, if and only if the unique key for two node objects are the same, then is_same_node will return true.

For namespace nodes, this returns a string "prefix:URI", which uniquely identifies a namespace.
