SYNOPSIS

    use ITS::WICS::FutureNodeManager;
    use ITS;
    my $f_manager = ITS::WICS::FutureNodeManager->new();
    my $ITS = ITS->new('xml', doc => 'myITSfile.xml');
    my ($ns) = $ITS->get_root->get_xpath('namespace::*');
    my $f_ns = create_future($ns);
    # change the document around, but don't delete any elements...
    $f_ns->new_node;

DESCRIPTION

This class saves nodes during DOM transformation so that they will still be in the document later. If you create a FutureNode for an attribute node, for example, it will remember its spot and create an element representing that attribute after you call the new_node method. This means that you can delete the original attribute without losing information. The document is not changed at all until you call new_node, so no XPath selectors are broken before then.

This is only guaranteed to work if document elements are not deleted after this object's creation. This is because some FutureNodes remember their location by their original parent, and deleting these will cause dereference errors.

EXPORTS

The following function may optionally be exported to the caller's namespace:

new_future

This is a convenience function for constructing an instance of this class (saves some typing, since the class has such a long name). The required arguments are the same as for new.

METHODS

new

Create a new FutureNode. The arguments are the FutureNodeManager which will manage this instance, the node to be represented, and the ITS::DOM instance containing it.

replace_el

If this FutureNode represents an element, this method trades out the element represented by this instance. Otherwise it croaks.

creates_element

True if the new_node method of this instance will cause the creation of a new element in the DOM, false otherwise.

type

Returns the type of node being represented (one of the strings returned by ITS::DOM::Node::type).

new_node

Ensures that the information in the contained node is represented in the HTML DOM. This may cause changes to the owning DOM (see creates_element). Calling this method multiple times, however, only changes the DOM once and always returns the same node.

Returns an ITS::DOM::Node object corresponding to the original Node (which might or might not be the same Node object).

name

Returns the name of the original node that this FutureNode represents. This is provided for sorting purposes.

new_path

Returns an XPath uniquely identifying the new node returned by new_node.
